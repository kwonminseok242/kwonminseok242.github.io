규제준수 리포팅, 리스크 스코어링, 이벤트 추적/감사, 무결성 검증

Project Overview

법정화폐 기반 스테이블 코인은 준비금(Reserve), 거래 흐름(Transaction), 규제 준수(Compliance) 관점의 리스크를 동시에 관리해야 합니다. 하지만 실무에서는 이벤트가 여러 시스템에 흩어져 있고, “왜 위험인지 / 어떤 근거로 판단했는지”를 빠르게 설명(Explain)하고 증빙(Proof)까지 남기기가 어렵습니다.

목표 증빙 자동화 리스크 가시화

본 프로젝트는 리스크 평가 → 규제준수보고서 생성 → 증빙팩(원본/무결성 정보 포함) 자동 생성까지 한 흐름으로 연결하는 리스크 관리 AI Agent를 구현했습니다. 사용자는 자연어로 질문하면, 에이전트가 필요한 데이터를 조회/요약하고, 근거와 함께 리포트를 구성하며, 결과물을 대시보드에서 바로 다운로드할 수 있도록 했습니다.

Core Tasks

  • 리스크 평가 질의/시나리오 설계 : 준비금 변동, 급격한 유출입, 비정상 패턴 등 리스크 분석 예시 질문 템플릿 구성
  • AI Agent 오케스트레이션 : LangGraph 기반으로 “질문 이해 → 근거 수집 → 분석/요약 → 리포트 생성” 단계형 플로우 구현
  • 규제준수 보고서 생성 : 리스크 요약, 근거(데이터 스냅샷/로그), 권고 액션을 포함한 보고서 포맷 자동 생성
  • 증빙팩 생성 파이프라인 : 이벤트 1건 또는 여러 거래를 묶어 JSON 증빙팩 생성 (원본/무결성 정보 포함) 및 다운로드 제공
  • 대시보드 연동 : 생성된 보고서/증빙팩 목록을 대시보드에서 조회하고 바로 다운로드 가능하도록 구성
  • 시각화/랭크 그래프 적용 : 리스크 항목 우선순위를 직관적으로 보여주기 위해 랭크 그래프를 ‘규제준수보고서’ 섹션에 집중 적용

Results & Learnings

결과물은 “리스크 판단의 일관성”과 “감사/대응을 위한 설명 가능성”을 강화하는 데 초점을 맞췄습니다. 단순히 답변을 내는 챗봇이 아니라, 근거를 남기고(증빙) 재검증 가능한 형태로 산출물을 제공하도록 설계했습니다.

  • 산출물 표준화 : 리스크 분석 결과를 보고서 템플릿으로 구조화하고, 동일한 질문에 대해 일관된 포맷으로 응답
  • 감사 대응성 강화 : 증빙팩에 원본 데이터/무결성 정보를 포함하여 “왜 이렇게 판단했는가”를 추적 가능하게 구성
  • Agent 설계 역량 : LangGraph로 단계/조건 분기/툴 호출을 관리하며, 실무형 에이전트 워크플로우를 구현
  • UI/커뮤니케이션 : 대시보드에서 생성물 목록 확인/다운로드까지 이어지도록 사용자 동선을 명확히 정리

개인적으로 배운 점
“정답 생성”보다 중요한 건, 금융/컴플라이언스 문맥에서 근거(데이터) → 판단(로직) → 증빙(무결성)이 한 세트로 묶여야 한다는 점이었습니다. 그래서 에이전트의 출력은 항상 재현 가능하고 검증 가능하도록 설계하는 방향으로 발전시켰습니다.
